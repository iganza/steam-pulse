* Running a Query on the CLI

** One-shot (start + poll)

#+begin_src bash
# 1. Start query — prints a query-id
aws logs start-query \
  --region us-west-2 \
  --log-group-names \
    '/steampulse/staging/crawler' \
    '/steampulse/staging/spoke/us-west-2' \
    '/steampulse/staging/ingest' \
  --start-time $(date -v-1H +%s) \
  --end-time   $(date +%s) \
  --query-string 'fields @timestamp, message | filter appid = 934700 | sort @timestamp asc'

# 2. Poll results (re-run until status = "Complete")
aws logs get-query-results --region us-west-2 --query-id <query-id>
#+end_src

** Wrapper script (start + poll automatically)

#+begin_src bash
query_logs() {
  local query="$1"
  local groups="${2:---log-group-names /steampulse/staging/crawler /steampulse/staging/spoke/us-west-2 /steampulse/staging/ingest}"
  local since="${3:-1h}"

  case "$since" in
    *m) start=$(date -v-${since}M +%s) ;;
    *h) start=$(date -v-${since%h}H +%s) ;;
    *d) start=$(date -v-${since%d}d +%s) ;;
    *)  start=$(date -v-1H +%s) ;;
  esac

  qid=$(aws logs start-query --region us-west-2 $groups \
    --start-time $start --end-time $(date +%s) \
    --query-string "$query" \
    --query 'queryId' --output text)

  echo "Query ID: $qid — polling..."
  while true; do
    result=$(aws logs get-query-results --region us-west-2 --query-id "$qid")
    status=$(echo "$result" | python3 -c "import json,sys; print(json.load(sys.stdin)['status'])")
    if [ "$status" = "Complete" ]; then
      echo "$result" | python3 -c "
import json, sys
d = json.load(sys.stdin)
for row in d['results']:
    print('  '.join(f['value'] for f in row if f['field'] != '@ptr'))
"
      break
    fi
    sleep 2
  done
}

# Example:
query_logs 'fields @timestamp, message, appid | filter appid = 934700 | sort @timestamp asc' '' '30m'
#+end_src

** Live tail (streaming — not Insights, but best for active monitoring)

~aws logs tail~ streams new events as they arrive. Use ~--filter-pattern~ (CloudWatch filter syntax, not Insights).

#+begin_src bash
# Tail all crawl services live, filter to one appid
aws logs tail \
  --region us-west-2 \
  --follow \
  --format short \
  --filter-pattern '{ $.appid = 934700 }' \
  '/steampulse/staging/crawler'

# aws logs tail only accepts ONE log group — run in parallel for multi-group:
aws logs tail --region us-west-2 --follow --format short \
  --filter-pattern '{ $.appid = 934700 }' \
  '/steampulse/staging/crawler' &
aws logs tail --region us-west-2 --follow --format short \
  --filter-pattern '{ $.appid = 934700 }' \
  '/steampulse/staging/spoke/us-west-2' &
aws logs tail --region us-west-2 --follow --format short \
  --filter-pattern '{ $.appid = 934700 }' \
  '/steampulse/staging/ingest' &
wait
#+end_src

Or use the repo's tail script (colour-coded, multi-group):
#+begin_src bash
poetry run python scripts/tail.py all --env staging
#+end_src

* Query Library

All queries: adjust the ~--start-time~ window as needed (last 1h, 6h, 1d, etc.).
Log groups to include are noted per query.

** 1. Follow a single game through the full crawl pipeline

Log groups: crawler + spoke + ingest

#+begin_src
fields @timestamp, @log, appid, message, task, count, reason,
       cursor, target, remaining, upserted, error
| filter appid = 934700
| sort @timestamp asc
#+end_src

Shows: message dispatch → spoke fetch → ingest upsert → re-queue or completion.
Set ~@log~ to see which service each line came from.

** 2. Crawl completion events (exhausted / early_stop / target_hit)

Log groups: ingest

#+begin_src
fields @timestamp, appid, message, reason, total
| filter message = "Reviews complete"
| sort @timestamp desc
| limit 100
#+end_src

~reason~ is one of: ~exhausted~, ~early_stop~, ~target_hit~.

** 3. Spoke fetch results — catch premature exhaustion

Log groups: spoke

#+begin_src
fields @timestamp, appid, count, exhausted, next_cursor
| filter message = "Fetched reviews"
| sort @timestamp desc
| limit 100
#+end_src

Flag: if ~exhausted = 1~ and ~count~ is suspiciously low (< 500), the cursor chain died early.

** 4. Review ingest volume by game (last N hours)

Log groups: ingest

#+begin_src
stats sum(upserted) as total_upserted by appid
| filter ispresent(upserted)
| sort total_upserted desc
| limit 50
#+end_src

** 5. All errors across the crawl pipeline

Log groups: crawler + spoke + ingest

#+begin_src
fields @timestamp, @log, appid, level, message, error
| filter level = "ERROR" or level = "WARNING"
| sort @timestamp desc
| limit 200
#+end_src

** 6. Steam API errors (rate-limits, failures)

Log groups: spoke

#+begin_src
fields @timestamp, appid, message, error
| filter message like "Steam"
| filter level = "ERROR" or level = "WARNING"
| sort @timestamp desc
| limit 100
#+end_src

** 7. Games crawled in the last window (activity summary)

Log groups: ingest

#+begin_src
stats count() as batches,
      sum(upserted) as reviews_upserted,
      max(@timestamp) as last_seen
by appid
| filter ispresent(upserted)
| sort last_seen desc
| limit 50
#+end_src

** 8. Re-queue hops for a game (shows cursor chain progress)

Log groups: ingest

#+begin_src
fields @timestamp, appid, message, remaining, total_so_far
| filter appid = 934700
| filter message = "Re-queued for next batch"
| sort @timestamp asc
#+end_src

Watch ~remaining~ count down toward zero.

Log groups: crawler + spoke + ingest + api + analysis

| sort @timestamp desc
| limit 50
