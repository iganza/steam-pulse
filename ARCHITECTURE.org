* Overview

SteamPulse is an AI-powered Steam game intelligence platform. It crawls the
Steam catalog, fetches reviews at scale using a multi-region spoke architecture,
synthesises them with Claude via AWS Bedrock, and serves structured reports
publicly via a Next.js frontend backed by a FastAPI Lambda.

All implementation decisions must be consistent with the flows defined here.
When a flow changes, update this file first. Use the drift checklist in each
flow to validate code alignment.

See also: =CLAUDE.md= for coding conventions, =steampulse-design.org= for
original design rationale.

** Flow-02: Metadata Crawl

1. After game-discovered, a game-discovered event should be published.
2. The MetadataEnrichmentQ should receive this event
3. The CrawlerFn should receive the event and process it, causing the game's metadata to be fetched and saved
4. On successful fetch and save of metadata, the "game-metadata-ready" event should be published to GameEventsTopic

- =GameRepository.upsert()= uses =ON CONFLICT DO UPDATE= (full metadata overwrite is correct here)
- S3 object is deleted after successful ingest (no orphaned objects)

** Flow-03: Review Crawl

#+BEGIN_SRC
Invariant: All English reviews for an eligible game are fetched and stored.
           If Steam returns a next_cursor, the game is re-queued until
           next_cursor is None (true exhaustion) or all batch reviews predate
           reviews_completed_at (early-stop on re-crawl).
           Re-crawls always restart from cursor="*" (newest-first) and use
           reviews_completed_at as an early-stop boundary to avoid re-fetching
           the entire review history.
           Duplicate review_ids are upserted (not duplicated) in DB.
#+END_SRC

#+BEGIN_SRC

      │                      │  │  completed_at = batch boundary (early_stop)    │  │
      │                      │  │              or NOW()           (exhausted)    │  │
      │                      │  │  early_stop = min(batch.timestamp_created)     │  │
      │                      │  │    < reviews_completed_at                      │  │
      │                      │  └─────────────────────────────────────┘  │
      │                      │                     │                    │
      │                      │  S3 delete object   │                    │
#+END_SRC

*Drift checklist:*
- =filter="recent"= in =steam_source.get_reviews()= (NOT =filter="all"=)
- =ReviewRepository.bulk_upsert()= uses =ON CONFLICT (steam_review_id) DO UPDATE= (update, not skip)
- Cursor is read from =app_catalog.review_cursor= before dispatch, not from SQS message
- Re-queue sends only ={appid}= to ReviewCrawlQ — cursor is stored in DB, not in message
- Ingest is data-only: no SNS events, no Step Functions trigger
- =mark_reviews_complete()= sets =review_cursor=NULL= and =reviews_completed_at=NOW()= atomically
- Early-stop: if =min(batch.timestamp_created) < reviews_completed_at=, treat as exhausted
- =review_status= does not exist on =app_catalog= — eligibility enforced at SNS filter level
- S3 object deleted AFTER cursor management and re-queue (safe for retry)
- =ensure_stub()= in game_repo uses =ON CONFLICT DO NOTHING= (must not overwrite real game names)

- State machine type is EXPRESS (not STANDARD) — no execution history, cheaper
- =ReportRepository.upsert()= overwrites on conflict (re-analysis is always a full replace)
