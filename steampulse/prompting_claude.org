Starting a Phase

   # Start Claude Code in the repo root
   claude

   # Then paste this opener:

   Read steam_analyzer_prompt.md and CLAUDE.md fully. Then implement Phase 0 only. Stop when done and show me the verification commands to run.

  Claude will build, then pause. You run the checks, confirm it works, then continue.

  Phase-by-Phase Prompts

  For each subsequent phase:

   Phase 0 is done and verified. Now implement Phase 1: FastAPI API Layer from steam_analyzer_prompt.md. Make no changes outside Phase 1 scope.

  The "stop and verify" pattern is key — don't let it barrel through multiple phases.

  When It Goes Off-Track

   Stop. Reread the constraints in CLAUDE.md → "Do Not Build" section. Revert [file] and try again staying within Phase 1 scope.

  Use Your Slash Commands Mid-Session

   /check-schema     # after any analyzer.py changes
   /cdk-diff         # before any infra deploy
   /analyze-game 440 # smoke test after Phase 1

  Key Tips

   - One phase per session — Claude Code has a context limit; fresh sessions stay sharp
   - Commit between phases — git commit after each phase passes verification
   - If it hallucinates a library: That package isn't in pyproject.toml. Use only existing deps or ask me before adding.
   - Resume after interruption: claude --continue picks up the last conversation

  The phased spec + CLAUDE.md + slash commands give Claude everything it needs. Your job is just to be the verification gate between phases.
