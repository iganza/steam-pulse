* Overview

SteamPulse uses two distinct Steam API surfaces:

- *Steam Web API* (~api.steampowered.com~) — requires an API key, used for catalog discovery
- *Steam Store API* (~store.steampowered.com~) — public/no key, used for game metadata and reviews

Both are called from ~src/lambda-functions/lambda_functions/crawler/~ and
~src/library-layer/library_layer/steam_source.py~.

* API 1 — GetAppList (Steam Web API)

** Endpoint

#+BEGIN_EXAMPLE
GET https://api.steampowered.com/IStoreService/GetAppList/v1/
#+END_EXAMPLE

Requires ~STEAM_API_KEY~ environment variable. Used during catalog refresh to
discover all Steam app IDs.

** Request Parameters

| Parameter      | Value       | Notes                               |
|----------------+-------------+-------------------------------------|
| ~key~          | ~$STEAM_API_KEY~ | Required                       |
| ~max_results~  | ~50000~     | Batch size per page                 |
| ~include_games~| ~1~         | Include games in results            |
| ~last_appid~   | ~<int>~     | Cursor for next page (pagination)   |

Pagination is cursor-based: iterate until ~have_more_results~ is false.

** Full Response Shape

#+BEGIN_SRC json
{
  "response": {
    "apps": [
      { "appid": 440, "name": "Team Fortress 2" },
      { "appid": 570, "name": "Dota 2" }
    ],
    "have_more_results": true,
    "last_appid": 1234567
  }
}
#+END_SRC

** Fields Extracted & Stored

Stored in the ~app_catalog~ table (orchestration/tracking only).

| API Field              | DB Column            | DB Table      | Notes                          |
|------------------------+----------------------+---------------+--------------------------------|
| ~response.apps[].appid~ | ~appid~             | ~app_catalog~ | Primary key                    |
| ~response.apps[].name~  | ~name~              | ~app_catalog~ | Raw name, not slugified        |
| /(discovery time)/     | ~discovered_at~      | ~app_catalog~ | Set to ~NOW()~ on insert       |
| /(default)/            | ~meta_status~        | ~app_catalog~ | Set to ~'pending'~ on insert   |
| /(default)/            | ~review_status~      | ~app_catalog~ | Set to ~'pending'~ on insert   |

Everything else in the response is discarded at this stage.

* API 2 — App Details + Reviews (Steam Store API)

** 2a. App Details

*** Endpoint

#+BEGIN_EXAMPLE
GET https://store.steampowered.com/api/appdetails?appids=<appid>&l=english
#+END_EXAMPLE

No API key required. Called during the app-crawl phase (one call per game).

*** Request Parameters

| Parameter | Value      | Notes                       |
|-----------+------------+-----------------------------|
| ~appids~  | ~"440"~    | Single app ID (as string)   |
| ~l~       | ~"english"~| Language for text fields    |

** 2b. Review Summary

*** Endpoint

#+BEGIN_EXAMPLE
GET https://store.steampowered.com/appreviews/<appid>
#+END_EXAMPLE

Called with ~num_per_page=1~ and ~language=all~ immediately after app details,
to get aggregate counts without fetching full review text.

*** Request Parameters

| Parameter      | Value   | Notes                                 |
|----------------+---------+---------------------------------------|
| ~json~         | ~"1"~   | Return JSON                           |
| ~num_per_page~ | ~"1"~   | Minimal — only need summary totals    |
| ~language~     | ~"all"~ | All languages for accurate total count|
| ~purchase_type~| ~"all"~ | Include all purchase types            |

*** Response Shape (relevant excerpt)

#+BEGIN_SRC json
{
  "success": 1,
  "query_summary": {
    "num_reviews": 1,
    "review_score": 9,
    "review_score_desc": "Overwhelmingly Positive",
    "total_positive": 186823,
    "total_negative": 8420,
    "total_reviews": 195243
  }
}
#+END_SRC

*** Fields Extracted & Stored (into games table)

| API Field                           | DB Column           | Notes                                  |
|-------------------------------------+---------------------+----------------------------------------|
| ~query_summary.total_reviews~       | ~review_count~      |                                        |
| ~query_summary.total_positive~      | ~total_positive~    |                                        |
| ~query_summary.total_negative~      | ~total_negative~    |                                        |
| /(derived)/                         | ~positive_pct~      | ~total_positive / total_reviews * 100~ |
| ~query_summary.review_score_desc~   | ~review_score_desc~ | Human label e.g. "Overwhelmingly Positive" |

Review count also written to ~app_catalog.review_count~ to drive the
decision of whether to enqueue the game for a full review crawl
(threshold: ≥ 500 reviews).

** 2c. Reviews (Paginated)

*** Endpoint

#+BEGIN_EXAMPLE
GET https://store.steampowered.com/appreviews/<appid>
#+END_EXAMPLE

Same endpoint as summary but with different parameters. Called iteratively
until all reviews are fetched or the cursor loops back to ~"*"~.

*** Request Parameters

| Parameter      | Value      | Notes                                    |
|----------------+------------+------------------------------------------|
| ~json~         | ~"1"~      | Return JSON                              |
| ~filter~       | ~"all"~    | Fetch all reviews (not just recent)      |
| ~language~     | ~"english"~| English only for LLM analysis            |
| ~num_per_page~ | ~"100"~    | Batch size per request                   |
| ~cursor~       | ~"*"~      | Starting cursor; updated each page       |
| ~purchase_type~| ~"all"~    | Include all purchase types               |

Rate limit handling: up to 6 retries, exponential backoff
~min(2^attempt + random(1,5), 120)s~, plus ~random(1.5, 3.5)s~ jitter
between each request.

*** Fields Extracted & Stored

Stored in the ~reviews~ table. The vast majority of response fields are
discarded.

| API Field                      | DB Column         | DB Table   | Notes                                |
|--------------------------------+-------------------+------------+--------------------------------------|
| ~review.recommendationid~      | ~steam_review_id~ | ~reviews~  | UNIQUE; used for upsert deduplication |
| /(appid from context)/         | ~appid~           | ~reviews~  | FK → games                           |
| ~review.voted_up~              | ~voted_up~        | ~reviews~  | BOOLEAN                              |
| ~review.author.playtime_at_review~ | ~playtime_hours~ | ~reviews~ | Converted: minutes ÷ 60             |
| ~review.review~                | ~body~            | ~reviews~  | Full review text                     |
| ~review.timestamp_created~     | ~posted_at~       | ~reviews~  | Unix → TIMESTAMPTZ                   |
| /(crawl time)/                 | ~crawled_at~      | ~reviews~  | Set to ~NOW()~ on insert             |

*Discarded:* author profile fields (~steamid~, ~num_games_owned~, etc.),
vote counts (~votes_up~, ~votes_funny~, ~weighted_vote_score~),
~steam_purchase~, ~received_for_free~, ~written_during_early_access~,
~hidden_in_steam_china~, and ~timestamp_updated~.

* Review Crawl Eligibility Thresholds

Only games above these deltas (new reviews since last crawl) are re-queued:

| Total Reviews       | Delta Required |
|---------------------+----------------|
| < 200               | ≥ 25           |
| 200 – 2,000         | ≥ 150          |
| 2,000 – 20,000      | ≥ 500          |
| 20,000 – 200,000    | ≥ 2,000        |
| > 200,000           | ≥ 10,000       |

Games with fewer than 500 total reviews are marked
~review_status='ineligible'~ and never crawled for reviews.
