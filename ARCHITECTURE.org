* Overview

SteamPulse is an AI-powered Steam game intelligence platform. It crawls the
Steam catalog, fetches reviews at scale using a multi-region spoke architecture,
synthesises them with Claude via AWS Bedrock, and serves structured reports
publicly via a Next.js frontend backed by a FastAPI Lambda.

All implementation decisions must be consistent with the flows defined here.
When a flow changes, update this file first. Use the drift checklist in each
flow to validate code alignment.

See also: =CLAUDE.md= for coding conventions, =steampulse-design.org= for
original design rationale.

** Flow-03: Review Crawl

#+BEGIN_SRC
Invariant: All English reviews up to the configured cap (REVIEW_LIMIT) are
           fetched and stored. Workflow state (cursor, remaining budget,
           start timestamp) is carried entirely in SQS message bodies —
           no DB reads during dispatch. The chain terminates when:
             (a) Steam returns next_cursor=None (exhaustion), OR
             (b) min(batch.timestamp_created) < reviews_completed_at (early-stop
                 on re-crawl — avoids re-fetching the entire review history), OR
             (c) remaining budget (target) ≤ batch count (target_hit).
           All three paths call mark_reviews_complete(), which sets
           reviews_completed_at and enables early-stop on the next re-crawl.
           Duplicate review_ids are upserted (not duplicated) in DB.
           Overshoot prevention: spoke uses limit=min(target, BATCH_SIZE)
           so the final batch never fetches more than the remaining budget.
#+END_SRC

*Steps:*

1. =GameEventsTopic= emits =game-metadata-ready= (=is_eligible=true=) → SNS filter
   routes to =ReviewCrawlQ=. Fresh-start message body: ={appid}= only.
2. =CrawlerFn= receives SQS record. Unwraps SNS envelope if present.
   Reads =cursor=, =target=, =started_at= from message body (no DB reads):
   - =cursor=: missing/null/"" → normalize to ="*"= (fresh start)
   - =target=: missing → =REVIEW_LIMIT=; =0= → skip dispatch (budget exhausted); positive → use as-is
   - =started_at=: missing → =datetime.now(UTC)=
3. =CrawlerFn= sends =ReviewSpokeRequest{appid, task:"reviews", cursor, target, started_at}=
   to the per-spoke SQS queue via =sqs.send_message()=.
   Routing: =appid % len(spoke_targets)= — deterministic, same appid always hits same spoke.
4. =SpokeCrawlerFn= receives the message via SQS event source mapping (=batch_size=1=, =max_concurrency=3=).
   Calls Steam =GET /appreviews/{appid}?filter=recent&cursor={cursor}&num_per_page=min(target,BATCH_SIZE)=.
   Overshoot prevention: =limit = min(target, BATCH_SIZE)= caps the final partial batch.
5. =SpokeCrawlerFn= gzips the review batch, writes to S3:
   =spoke-results/reviews/{appid}-{uuid}.json.gz=
6. =SpokeCrawlerFn= sends =ReviewSpokeResult{appid, task, success, s3_key, count, next_cursor, target, started_at, spoke_region}=
   to =SpokeResultsQ=. =target= and =started_at= are passed through unchanged for the ingest hop.
7. =SpokeIngestFn= receives result from =SpokeResultsQ=. Fetches S3 object, decompresses.
   Calls =ReviewRepository.bulk_upsert()= — =ON CONFLICT(steam_review_id) DO UPDATE=.
8. =SpokeIngestFn= evaluates termination (precedence: exhausted → early_stop → target_hit → re-queue):
   - =exhausted=: =msg.next_cursor is None=
   - =early_stop=: =reviews_completed_at IS NOT NULL= AND =min(batch.timestamp_created) < reviews_completed_at=
   - =target_hit=: =msg.target is not None= AND =msg.target ≤ msg.count=
   - else: re-queue with =new_remaining = msg.target - msg.count=
9. On exhausted/early_stop/target_hit: =CatalogRepository.mark_reviews_complete(appid, completed_at)=
   sets =reviews_completed_at= — the only DB write in the whole chain.
   =completed_at=: batch boundary timestamp on early_stop; =NOW()= on exhausted/target_hit.
10. On re-queue: =SQS.send_message= directly to the per-spoke crawl queue with
    =ReviewSpokeRequest{appid, cursor:next_cursor, target:new_remaining, started_at}=.
    This bypasses the primary crawler — loop returns directly to step 4.
11. S3 object deleted after all DB/SQS work succeeds (safe to retry on failure — object still present).

*Drift checklist:*
- Dispatch is via SQS =send_message= to per-spoke crawl queue (NOT Lambda async invoke)
- Spoke consumes via SQS event source mapping (=batch_size=1=, =max_concurrency=3=)
- =filter="recent"= in =steam_source.get_reviews()= (NOT =filter="all"=)
- =ReviewRepository.bulk_upsert()= uses =ON CONFLICT (steam_review_id) DO UPDATE= (update, not skip)
- Cursor, target, and started_at are read from the SQS message body — no DB reads during dispatch
- Fresh-start message body contains only ={appid}=; cursor/target/started_at are normalized by CrawlerFn
- =cursor= normalization: missing/null/"" → ="*"= (fresh start)
- =target= normalization: missing → =REVIEW_LIMIT=; =0= → skip dispatch; positive → use as-is
- Re-queue from ingest goes directly to per-spoke SQS queue (bypasses CrawlerFn entirely)
- Re-queue sends =ReviewSpokeRequest{appid, cursor, target:remaining, started_at}= — =target= decremented by =msg.count= each hop
- =target_hit= check: =msg.target ≤ msg.count= (remaining budget ≤ this batch — NOT total DB count vs original total)
- Overshoot prevention: spoke computes =limit = min(target, BATCH_SIZE)= before calling Steam
- Ingest is data-only: no SNS events, no Step Functions trigger from this flow
- =mark_reviews_complete()= sets only =reviews_completed_at= — no cursor column (dropped in migration 0008)
- Early-stop boundary: =completed_at = min(batch.timestamp_created)= on early_stop; =NOW()= on exhausted/target_hit
- S3 object deleted AFTER all DB writes and SQS re-queue succeed (safe for retry on failure)
- =reviews_completed_at= is the sole termination signal — no =review_cursor= or =reviews_target= columns
