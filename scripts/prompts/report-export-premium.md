# Report Export — HTML & PDF Download (Premium Feature)

## Overview

Allow authenticated premium users to export any game report as a clean HTML file
or a print-ready PDF. This is a key value prop for indie developers doing competitive
research — they want to share reports with teammates or save them offline.

## Access Control

Export is **premium-only**. Gate behind the same API key / session check used by
`/api/validate-key`. Free users see the export buttons as locked/disabled with a
"Pro feature" tooltip pointing to the upgrade page.

No export for anonymous users. No watermark needed — the report footer already has
the SteamPulse branding and URL.

## What Gets Exported

The full report — all sections including the premium-gated ones:
- `dev_priorities`, `churn_triggers`, `player_wishlist` (normally paywalled)
- All public sections: sentiment, friction, strengths, competitive context, etc.
- Game metadata: name, appid, cover art, review count, analysis date
- A footer: "Generated by SteamPulse · steampulse.io · [date]"

Exclude: ads, nav, header, CTAs, request-analysis UI. Export is clean content only.

## HTML Export

- Endpoint: `GET /api/export/[appid]/report.html` (auth required)
- Returns a self-contained HTML file with inline CSS (no external deps)
- Styled to match the SteamPulse report design, readable without the site
- `Content-Disposition: attachment; filename="[game-slug]-report.html"`
- The inline CSS should be minimal and print-friendly (good contrast, readable fonts)

## PDF Export

Two implementation options — pick the simpler one:

**Option A (recommended): Browser print via CSS** 
- Add a `?print=1` query param to the report page that renders a print-optimized layout
- Add a `@media print` stylesheet that hides nav/ads/CTAs and formats for A4/Letter
- The "Export PDF" button opens `window.open('/games/[appid]/[slug]?print=1')` and
  triggers `window.print()` via a small script
- Zero server-side dependencies — works everywhere, no Puppeteer/headless Chrome

**Option B: Server-side PDF via Puppeteer/Playwright**
- Lambda renders the report page headlessly and returns a PDF blob
- More reliable formatting but adds cold start latency and Lambda size
- Only worth it if Option A produces unacceptable output

Start with Option A. Add Option B only if users complain about PDF quality.

## UI

- On the report page, add an "Export" button in the report header (top-right area)
- Dropdown with two options: "Download HTML" and "Export PDF"
- If user is not premium: buttons are visible but disabled, tooltip says "Pro feature"
  with a link to the upgrade/pro page
- If user is premium: clicking "Download HTML" fetches `/api/export/[appid]/report.html`
  and triggers browser download; clicking "Export PDF" opens print dialog

## API Implementation

```
GET /api/export/{appid}/report.html
  - Auth: validate API key from Authorization header or cookie
  - Fetch report JSON from ReportRepository
  - Render to self-contained HTML using a Jinja2-style template (or f-string in Python)
  - Return with Content-Type: text/html, Content-Disposition: attachment
  - 401 if not authenticated, 403 if not premium, 404 if no report exists
```
